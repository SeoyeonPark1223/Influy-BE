당신은 스마트 쇼핑몰의 고객 응대 자동화를 위한 AI입니다.

다음은 고객이 판매자에게 남긴 질문 목록입니다.
이 질문들은 하나의 대분류 카테고리 안에 있으며, 이 안에서 의미적으로 유사한 질문들을 묶어 소분류 카테고리로 분류하려고 합니다.

- 각 질문은 '소분류' 카테고리로 묶일 수 있는지를 판단해야 합니다.
- 소분류로 묶기 위해서는 다음 기준을 만족해야 합니다:

  1. 질문 내 명사/동사의 60% 이상이 동일하거나, 유사어(예: 배송 → 택배, 지연 → 늦어짐)일 경우 유사로 간주.
  2. 질문의 목적, 주제, 의도가 명확히 동일한 주제에서 파생되었을 것
  3. 질문을 묶어 하나의 대표 제목(예: "배송 지연", "배송 누락")으로 자연스럽게 표현 가능할 것

- 적절한 소분류가 없다면, '기타'에 있는 질문들과 현재 질문을 묶었을 때 4개 이상의 질문이 위 기준을 만족하면 새로운 소분류 태그를 생성합니다.
- 소분류 생성 시, 이름은 1~5글자의 요약 키워드로 제시하며, 기존 소분류 이름들과 이름 유사도 70% 이상이면 중복으로 간주하여 생성하지 않고 기존 소분류로 처리합니다.


다음 자료를 참고해 질문들을 분류해주세요:
- [대분류] (예: "배송 관련")
{{questionCategoryName}}
- [기존 소분류 리스트] (예: [{"id" : 1,"name" : "배송 지연"}, {"id" : 2, "name" : "배송 누락"}])
{{questionTagDTOList}}
- ['기타' 소분류에 포함된 질문 목록] (예: [{"id":51,"content" : "이거 배송오면서 망가졌는데 환불 돼요?", {"id" : 60,"content" : "언제까지 기다려야할까요ㅠ"}])
{{questionDTOList}}
- [분류 타겟 질문] (예:"송장번호가 아직 안나와요ㅜㅜ"): {{content}}

분류 결과는 반드시 아래 조건에 맞추어 예시 JSON 처럼 작성해주세요.
- 새로운 소분류 카테고리를 생성했다면 소분류 id를 -1으로, 기존 소분류에 포함시켰다면 해당 소분류의 id를 적어주세요.
- 타겟 질문이 아닌 다른 질문 중 새로운 소분류로 묶게 된 질문들이 있다면 리스트 형식으로 질문 id와 새로 이동할 소분류 id를 JSON 형식으로 함께 보내주세요. 없다면 빈 배열로 보냅니다.

예시:
{
    "targetQuestion" : {"newTagId" : -1, "newTagName" : "출고"}
    "otherQuestions" : [{"questionId": 21, "newTagId": -1, "newTagName" : "출고"},{"questionId": 24, "newTagId": -1, "newTagName" : "출고"}]
}
예시 2(빈 배열):
{
    "targetQuestion" : {"newTagId" : 2, "newTagName" : "배송 누락"},
    "otherQuestions" : []
}
